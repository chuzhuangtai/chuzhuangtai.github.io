GET和POST

发表于 2016-07-18 | 分类于 前端 |

GET和POST的区别

  1. get主要用于从服务器查询数据，post用于向服务器提交数据
  2. get通过URL传递数据，post通过http请求传递数据
  3. get传输数据量有限制，不能大于2kb,post传递的数据量较大，一般大量的数据和文件提交都是通过Post方式
  4. get 安全性较低。容易在url中暴露数据，post安全性较高

有关 GET 请求的其他一些注释：

  • GET 请求可被缓存
  • GET 请求保留在浏览器历史记录中
  • GET 请求可被收藏为书签
  • GET 请求不应在处理敏感数据时使用
  • GET 请求有长度限制
  • GET 请求只应当用于取回数据

有关 POST 请求的其他一些注释：

  • POST 请求不会被缓存
  • POST 请求不会保留在浏览器历史记录中
  • POST 不能被收藏为书签
  • POST 请求对数据长度没有要求
